# Qinghe - 安卓弱网测试工具

一款基于VPNService实现的Android弱网测试工具，支持模拟各种网络环境。

## 功能特性

- 基于VPNService实现核心弱网模拟功能
- 支持选择指定应用进行弱网测试
- 支持自定义弱网配置（延时、抖动、带宽限制、丢包等）
- 配置本地永久存储，支持编辑和删除
- 悬浮窗快速切换配置，即时生效
- 兼容Android 5.0至最新系统

## 配置参数

| 参数 | 单位 | 默认值 | 说明 |
|------|------|--------|------|
| 上行/下行延时 | ms | 0 | 网络延迟 |
| 上行/下行延时抖动 | ms | 0 | 延迟波动范围 |
| 上行/下行带宽 | kbps | 0 | 带宽限制(0为不限制) |
| 上行/下行随机丢包 | % | 0 | 随机丢包概率(0-100) |
| 上行/下行连续丢包放行时长 | ms | 0 | 连续丢包模式放行时间 |
| 上行/下行连续丢包丢包时长 | ms | 0 | 连续丢包模式丢包时间 |
| 弱网控制协议 | - | 3 | 1=TCP, 2=UDP, 3=全部 |

## 弱网配置实现路径与排查建议

当前实现的关键链路如下：

1. `ConfigEditActivity`：录入弱网参数并保存到 Room 数据库。
2. `MainActivity`：选择配置后启动 `QingheVpnService`，并把 `config_id` 传给 VPN 服务。
3. `QingheVpnService`：读取活动配置后建立 TUN 接口，在 `runVpnLoop()` 中对命中的上行数据包应用延时、抖动、随机丢包、连续丢包和带宽限制。
4. `FloatingWindowService`：运行中切换配置时调用 `QingheVpnService.updateConfig()` 做热更新。

### 已知容易导致“配置后不生效”的场景

- **当前代码只真正消费了上行配置**：下行延时、下行带宽、下行随机丢包等字段虽然可以保存，但现有 `runVpnLoop()` 没有独立的下行转发链路，因此不会像 UI 描述那样生效。
- **VPN 数据面尚未接入真实远端转发**：目前服务从 TUN 读取数据后直接写回同一个 `vpnInterface`，没有建立到真实网络的 socket 转发链路，也没有对真实上游连接调用 `protect()`。这会导致很多 App 看起来已经“开启弱网”，但实际请求没有按预期经过完整的弱网模拟流程。
- **带宽限制对低带宽值非常敏感**：旧实现使用整数除法换算 `bytesPerMs`，在低带宽下可能直接算成 `0`，表现为“带宽限制完全不生效”。当前版本已改为按数据包位数计算最小等待时间。
- **运行中切换配置可能存在生效延迟**：悬浮窗切换配置时是异步读取数据库并更新内存配置，如果正在压测，建议暂停业务流量后再切换，避免把短暂的线程切换窗口误判为配置失效。

### 建议优先排查的点

1. **先确认链路是否真的进入 VPN**
   使用只经过系统网络栈的 App 做验证，并观察系统 VPN 图标、前台通知与目标 App 实际流量是否同时出现。

2. **优先验证已实现的上行能力**
   先只设置上行延时/上行随机丢包/上行带宽，不要一开始就混合下行参数或连续丢包参数，这样更容易判断是哪一段失效。

3. **不要把“下行参数无变化”直接当成参数传递 bug**
   现阶段更大的可能是实现路径尚未覆盖下行，而不是 Room/Intent 参数丢失。

4. **抓包或打日志确认关键字段**
   建议在 `MainActivity.startService()`、`QingheVpnService.onStartCommand()`、`QingheVpnService.updateConfig()` 和 `runVpnLoop()` 中打印 `config_id`、协议、延时、带宽、丢包率，确认保存值、启动值、运行值是否一致。

5. **关注平台差异**
   不同 Android 版本、厂商 ROM、后台限制策略、悬浮窗权限与 VPN 权限状态，都可能让服务看起来“启动成功”但实际没有持续运行。

## CI/CD

项目配置了GitHub Actions自动构建：
- 优先编译签名正式版APK
- 密钥校验失败或无密钥时自动编译测试版APK

| Secret | 说明 |
|--------|------|
| KEYSTORE_BASE64 | 签名文件Base64编码 |
| KEYSTORE_PASSWORD | 密钥库密码 |
| KEY_ALIAS | 密钥别名 |
| KEY_PASSWORD | 密钥密码 |
